## 1. 背景与目标

### 1.1 业务背景

企业内部存在10+个业务系统，每个系统平均有200页的操作手册。当前这些手册以Word文档形式分散存储，由IT支持团队维护。一线客服人员（100人）在解答用户问题时，需要手动翻阅文档查找操作步骤和错误码解决方案，效率低下且准确性难以保证。

手册更新频率为每季度到每年一次，更新时需要IT支持团队重新整理文档，内容维护成本高。同时，文档中包含大量截图和表格，传统检索方式难以精准定位问题。

### 1.2 问题陈述

**一线客服人员在解答终端客户操作问题时遇到查找困难、响应慢的问题，导致客户满意度低、客服效率低下，当前解决方案是人工翻阅Word文档，但存在检索耗时、难以定位精准步骤、错误码响应不及时、手册更新维护成本高的不足。**

### 1.3 目标

**核心目标**：构建AI驱动的系统操作知识库，实现操作指南和错误码的智能检索与自动生成，提升客服响应效率和问题解决率。

**量化指标**：
- 客服日均查询响应时间：从5分钟降低到30秒（目标提升10倍）
- 客服查询成功率：从70%提升到90%
- 系统覆盖度：支持10+个系统，单系统200页手册
- 并发支持能力：瞬时10个请求，100人日常使用
- 手册处理时间：单系统200页文档自动解析<10分钟

**时间目标**：上线后3个月内达成所有量化指标。

### 1.4 核心假设

1. **检索有效性假设**：AI基于向量检索能够准确匹配用户问题与手册内容，准确率>85%
2. **错误码识别假设**：系统能够自动识别文档中的错误码并关联到对应操作步骤，关联准确率>80%
3. **自动生成假设**：基于MPC/RAG能力能够自动生成表单内容，可用性>70%
4. **用户采纳假设**：客服人员愿意使用新系统替代原有文档查找方式，周活跃率>80%

## 2. 用户场景与需求

### 2.1 用户画像

#### 核心用户：一线客服人员

- 人口属性：25-35岁，高中/大专学历，一线和新一线城市
- 行为特征：
  - 使用频率：日均10次查询
  - 使用时段：9:00-18:00工作时段，高峰期14:00-16:00
  - 使用场景：接听客户电话、在线客服会话中快速查找操作指南
- 心理特征：
  - 核心诉求：快速、准确地找到操作步骤，减少客户等待时间
  - 痛点：文档分散、检索困难、错误码对应方案不明确
  - 动机：提升工作效率、减少客户投诉
- 技术能力：
  - 互联网熟练度：中等，熟悉基础办公软件
  - 产品接受度：愿意学习新工具，但要求操作简单

#### 次要用户：IT支持团队

- 人口属性：30-40岁，本科及以上学历，技术背景
- 行为特征：
  - 使用频率：每季度更新手册，日常修正数据标注
  - 使用场景：导入新系统手册、修正错误标注、维护知识库
- 心理特征：
  - 核心诉求：降低手册维护成本、提升知识库准确性
  - 痛点：手动拆分文档耗时、标注错误需人工修正

- **错误码查询**：通过错误码直接返回解决方案

- **动作类型识别**：自动识别动作类型（新增、修改、删除、审核、查询）

- **查询历史记录**：记录客服查询历史，方便快速复用

- **智能推荐**：根据查询历史推荐相关操作步骤

- **导出功能**：支持导出操作步骤为PDF

- **视频教程集成**（延后到V2.0）
- **社区问答**（延后到V2.0）

## 3. 功能详细说明

### 3.1 功能模块一：文档导入与智能解析

#### 功能描述

**输入**：Word格式操作手册文档

**处理**：
1. 文档图片处理：提取所有图片，上传阿里云OSS，文档中替换为图片URL
2. 文档表格处理：将表格转换为图片并上传OSS，替换为图片URL
3. 文档模块识别：识别文档中的模块标题，提取模块标识（URL），标记模块范围
4. 文档模块分割：按照提取的模块，将内容分割处理
5. 模块文档动作识别：识别模块下的动作标题，提取动作标识（URL），标记动作类型（新增/修改/删除/审核/查询）
6. 模块动作分割：按照动作将文档分割，动作过长时继续细分
7. 分割文档数据标记：标注系统、模块、动作，错误码打上"错误:错误码"标签
8. 数据向量化：将分割后的文本向量化存储到PGSQL

**输出**：结构化的知识库数据（系统→模块→动作→步骤）

#### 交互流程

```
1. IT支持人员进入"知识库管理"页面
2. 点击"导入系统"按钮
3. 上传Word文档（支持拖拽上传）
4. 系统自动解析文件名生成系统名称，支持手动修改
5. 系统开始解析，显示解析进度（模块识别→动作识别→向量化）
6. 解析完成后，展示解析结果预览（系统树形结构）
7. IT审核解析结果，可点击展开查看详细内容
8. 确认无误后，点击"完成导入"
9. 数据正式入库，知识库可用
```

#### 业务规则

- **系统名称生成规则**：优先从文件名提取，去除扩展名和特殊字符；提取失败则提示手动输入
- **模块识别规则**：识别文档中符合标题格式的段落（如"1.x"、"第x章"、"x.x"等），连续同一层级的标题属于同一模块
- **动作类型识别规则**：根据动作关键词自动识别类型：
  - "创建"、"新增"、"添加" → 新增
  - "修改"、"编辑"、"更新" → 修改
  - "删除"、"移除" → 删除
  - "审核"、"审批"、"通过" → 审核
  - "查询"、"搜索"、"查看"、"列表" → 查询
- **错误码识别规则**：识别符合错误码格式的文本（如ERR-xxxx、ERROR:xxx、错误码xxx等），标记为"错误:错误码"
- **动作排序规则**：按照文档中的出现顺序自动排序

#### UI规格

- **导入按钮**：位于知识库管理页面右上角，尺寸180x44px，蓝色背景，白色文字"导入系统"
- **上传区域**：支持拖拽上传，虚线边框区域，尺寸600x300px，居中显示"点击或拖拽上传Word文档"
- **进度显示**：解析过程中显示进度条，下方显示当前步骤（如"正在识别模块...（3/5）"）
- **结果预览**：树形结构展示，左侧为系统列表，右侧为选中系统的模块和动作详情
- **确认按钮**：位于页面底部右侧，尺寸120x40px，绿色背景，白色文字"完成导入"

#### 异常处理

- 异常1：上传非Word文件 → 提示"仅支持.doc或.docx格式文件"
- 异常2：文件大小超过50MB → 提示"文件大小不能超过50MB"
- 异常3：文档解析失败（无有效模块） → 提示"文档格式不符合要求，请检查文档结构"
- 异常4：OSS上传失败 → 提示"图片上传失败，请检查网络连接"
- 异常5：向量化失败 → 提示"知识库生成失败，请联系技术支持"

### 3.2 功能模块二：知识库标注修正

#### 功能描述

**输入**：自动解析后的知识库数据

**处理**：
1. 系统管理：修改系统名称、描述
2. 模块管理：新增、修改、转移模块动作
3. 模块动作管理：新增、修改、转移模块动作文档内容，对模块动作打标签

**输出**：更新后的知识库数据，向量数据同步更新

#### 交互流程

```
【系统管理】
1. 进入"知识库管理"页面
2. 在系统列表中选择要修改的系统
3. 点击"编辑系统"按钮
4. 修改系统名称、描述
5. 点击"保存"
6. 系统提示"保存成功"，向量数据同步更新

【模块管理】
1. 在系统详情页面，选择要管理的模块
2. 点击"管理模块"按钮
3. 可以新增子模块、修改模块名称、转移模块下的动作到其他模块
4. 点击"保存"
5. 系统提示"保存成功"，向量数据同步更新

【动作管理】
1. 在模块详情页面，选择要管理的动作
2. 点击"管理动作"按钮
3. 可以新增动作、修改动作名称、转移动作内容到其他动作
4. 可以修改动作标签（动作类型、错误码标签等）
5. 点击"保存"
6. 系统提示"保存成功"，向量数据同步更新
```

#### 业务规则

- **系统名称唯一性**：同一租户下系统名称不能重复
- **模块转移规则**：转移模块动作时，子模块和动作一并转移
- **动作转移规则**：转移动作内容时，保留原动作的基本信息，仅转移文档内容
- **向量同步规则**：修改标注信息后，异步更新对应的向量数据，确保检索一致性
- **删除规则**：删除系统/模块/动作时，级联删除下级内容，需二次确认

#### UI规格

- **编辑按钮**：每个系统/模块/动作右侧显示编辑图标，点击后弹出编辑弹窗
- **弹窗尺寸**：宽度600px，高度自适应，居中显示
- **表单布局**：表单项上下排列，标签宽度120px
- **转移操作**：提供下拉选择框，列出可转移的目标位置
- **保存按钮**：位于弹窗底部右侧，尺寸100x36px，蓝色背景

#### 异常处理

- 异常1：系统名称重复 → 提示"系统名称已存在，请更换"
- 异常2：转移目标无效 → 提示"不能转移到自身或子模块下"
- 异常3：向量同步失败 → 提示"数据已保存，但向量更新失败，请稍后重试"

### 3.3 功能模块三：客服查询操作指南

#### 功能描述

**输入**：客服选择的系统、模块、动作，或描述的自然语言问题

**处理**：
1. 前端传输当前模块标识及动作标识、问题描述
2. 系统通过RAG查询关联的向量内容
3. 返回匹配的操作步骤，包含文字说明和图片URL
4. 前端通过富文本展示操作步骤

**输出**：格式化的操作指南（文字+图片）

#### 交互流程

```
【方式一：结构化筛选】
1. 客服进入"操作指南查询"页面
2. 选择系统（下拉列表）
3. 选择模块（下拉列表，根据系统联动）
4. 选择动作（下拉列表，根据模块联动）
5. 点击"查询"按钮
6. 系统返回操作步骤，富文本展示
7. 如有错误码相关内容，同时展示错误码说明

【方式二：自然语言搜索】
1. 客服在搜索框输入问题描述（如"如何在客户管理创建客户"）
2. 点击搜索按钮
3. 系统通过向量检索匹配最相关的操作步骤
4. 返回Top 3结果，客服点击查看详情
```

#### 业务规则

- **筛选联动规则**：选择系统后，模块列表仅显示该系统下的模块；选择模块后，动作列表仅显示该模块下的动作
- **检索匹配规则**：基于向量相似度匹配，相似度阈值>0.7才返回结果
- **结果排序规则**：按相似度降序排列，Top 3结果展示
- **富文本展示规则**：文字保持原有格式，图片URL渲染为图片，支持缩放查看

#### UI规格

- **筛选区域**：位于页面顶部，三列下拉列表横向排列，每列宽度200px
- **搜索框**：位于筛选区域下方，宽度600px，占位符"请输入问题描述，如：如何在客户管理创建客户"
- **结果区域**：位于页面下方，卡片式展示，每张卡片宽度800px
- **富文本展示**：支持Markdown格式，图片最大宽度600px，点击放大查看

#### 异常处理

- 异常1：未选择系统/模块/动作 → 提示"请选择系统、模块和动作"
- 异常2：检索无结果 → 提示"未找到相关操作指南，请尝试调整搜索词"
- 异常3：图片加载失败 → 显示"图片加载失败，请刷新重试"

### 3.4 功能模块四：错误码查询与智能指导

#### 功能描述

**输入**：错误码

**处理**：
1. 通过错误码直接返回操作问题
2. 通过AI识别错误码所属的模块和动作
3. 给出操作指南

**输出**：错误说明+解决方案

#### 交互流程

```
【错误码查询】
1. 客服进入"错误码查询"页面
2. 输入错误码（如ERR-5003）
3. 点击"查询"按钮
4. 系统返回：
   - 错误码说明（系统定义的错误信息）
   - 所属模块和动作（AI识别）
   - 操作指南（解决步骤）
5. 客服参考指南指导客户
```

#### 业务规则

- **错误码格式规则**：支持常见错误码格式（ERR-xxxx、ERROR:xxx、xxx错误码等）
- **模糊匹配规则**：支持部分匹配，如输入"5003"也能匹配"ERR-5003"
- **AI识别规则**：基于错误码和向量检索，识别最可能的模块和动作，准确率目标>80%
- **关联展示规则**：如果错误码已关联到具体动作，直接展示该动作的操作指南；否则展示AI推荐的Top 2动作指南

#### UI规格

- **错误码输入框**：位于页面顶部，宽度400px，大字体显示
- **查询按钮**：位于输入框右侧，尺寸120x40px
- **结果展示**：分三部分展示
  - 错误说明：红色背景卡片，突出显示
  - 所属模块动作：蓝色标签展示
  - 操作指南：富文本展示

#### 异常处理

- 异常1：错误码未找到 → 提示"未找到该错误码，请检查输入是否正确"
- 异常2：AI识别失败 → 提示"无法确定该错误码的具体位置，请提供更多信息"

### 4.4 可维护性要求

- **日志记录**：记录关键操作日志，包括文档导入、查询、标注修改等

  - 文档解析失败率>10%时告警
  - 向量检索失败率>5%时告警
- **数据备份**：每日备份知识库数据和向量数据，保留30天

## 7. 风险与依赖

### 7.1 风险清单

| 风险描述 | 影响程度 | 发生概率 | 应对措施 |
|---------|---------|---------|---------|
| Word文档结构不规范，导致自动解析失败 | 高 | 中 | 1. 提供文档模板供IT参考 2. 解析失败时提供人工修正入口 3. 解析规则支持自定义配置 |
| 向量检索准确率不达标，影响客服使用体验 | 高 | 中 | 1. 引入混合检索（向量+关键词） 2. 上线前进行充分测试和调优 3. 提供人工反馈入口，持续优化模型 |
| 错误码识别准确率低，AI推荐不准确 | 中 | 中 | 1. 支持IT手动关联错误码到具体动作 2. 引入规则引擎兜底 3. 持续收集错误码查询反馈，优化算法 |
| 并发量超出预期，系统性能瓶颈 | 中 | 低 | 1. 提前进行压力测试 2. 设计水平扩展架构 3. 引入缓存层减轻数据库压力 |
| 阿里云OSS服务异常，影响图片展示 | 中 | 低 | 1. OSS设置跨区域备份 2. 本地缓存热门图片 3. 提供降级方案（不展示图片） |

### 7.2 外部依赖

- 依赖1：阿里云OSS → 用于存储文档中的图片和表格，需提前创建Bucket并配置访问权限
- 依赖2：PGSQL向量插件（pgvector）→ 需安装pgvector插件支持向量存储和检索
- 依赖3：Spring AI框架 → 用于集成RAG能力，需确认版本兼容性
- 依赖4：IT支持团队 → 提供操作手册文档，并进行标注修正工作
